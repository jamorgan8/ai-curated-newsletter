# Automated Daily News Report

A daily HTML news briefing generated by Gemini 2.5 Flash (free) with Google Search grounding, published to GitHub Pages, and delivered to your inbox each morning.

## What You Get

A clean, mobile-friendly news report covering:

1. **Federal Politics** — Top 3–5 congressional, executive, and Supreme Court stories (bipartisan framing)
2. **Tennessee & Nashville Politics** — Nashville city news, statewide TN updates, and notable stories from other states
3. **World Politics** — 3–4 international stories with a US foreign policy focus
4. **AI & Technology** — Latest model releases, company news, policy developments, and research breakthroughs
5. **Sports** — Tottenham Hotspur, Nashville SC, plus a dynamic roundup of the most active US leagues

Each morning you receive an email with a brief preview of top stories and a link to the full report on GitHub Pages.

## Setup Guide

### 1. Create a GitHub Repository

```bash
cd automated_news_report
git init
git add .
git commit -m "Initial commit: automated daily news report"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/automated_news_report.git
git push -u origin main
```

### 2. Get a Gemini API Key (Free)

1. Go to [aistudio.google.com/apikey](https://aistudio.google.com/apikey)
2. Sign in with your Google account
3. Click **Create API Key**
4. Copy the key — you'll add this as a GitHub secret in step 5

The free tier includes 15 requests/minute and 1,500 requests/day — far more than the 5 requests this report uses daily.

### 3. Generate a Gmail App Password

Gmail App Passwords let this script send email without exposing your main password.

1. Go to [myaccount.google.com](https://myaccount.google.com/)
2. Navigate to **Security**
3. Under "Signing in to Google," ensure **2-Step Verification** is turned ON
   - If it's off, click it and follow the setup flow
4. After 2-Step Verification is enabled, go to **Security → 2-Step Verification → App passwords**
   - Or go directly to [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
5. Enter an app name (e.g., `News Report`) and click **Create**
6. Copy the 16-character password (looks like `abcd efgh ijkl mnop`)
7. Save this — you'll add it as a GitHub secret in step 5

> **Note:** You must use this App Password, not your regular Gmail password. Regular passwords won't work with SMTP when 2FA is enabled.

### 4. Enable GitHub Pages

1. Go to your repo on GitHub
2. Navigate to **Settings → Pages**
3. Under "Source," select **Deploy from a branch**
4. Under "Branch," select `gh-pages` and `/ (root)`
5. Click **Save**

> The `gh-pages` branch will be created automatically the first time the workflow runs. You can set this up after the first run.

Your report will be available at: `https://YOUR_USERNAME.github.io/automated_news_report/`

### 5. Add GitHub Actions Secrets

1. Go to your repo on GitHub
2. Navigate to **Settings → Secrets and variables → Actions**
3. Click **New repository secret** and add each of these:

| Secret Name          | Value                                              |
| -------------------- | -------------------------------------------------- |
| `GEMINI_API_KEY`     | Your Gemini API key from step 2                     |
| `GMAIL_ADDRESS`      | Your full Gmail address (e.g., `you@gmail.com`)     |
| `GMAIL_APP_PASSWORD` | The 16-character App Password from step 3           |
| `TO_EMAIL`           | Recipient email address (can be the same Gmail)     |

### 6. Run a Test

1. Go to your repo on GitHub
2. Navigate to **Actions**
3. Click **Daily News Report** in the left sidebar
4. Click **Run workflow → Run workflow**
5. Watch the workflow run — it takes 2–3 minutes
6. Check your email and the GitHub Pages URL for the report

## Schedule

The workflow runs daily at **11:00 UTC**, which is:
- **6:00 AM CDT** (Central Daylight Time, March–November)
- **5:00 AM CST** (Central Standard Time, November–March)

The report typically generates in 2–3 minutes, so the email arrives by 6:05 AM CDT.

## Customization

### Change the delivery time

Edit the cron schedule in `.github/workflows/daily_report.yml`:

```yaml
schedule:
  - cron: "0 11 * * *"  # 11:00 UTC
```

Use [crontab.guru](https://crontab.guru/) to calculate your desired UTC time. GitHub Actions uses UTC only.

### Change report topics

Edit the section prompts in `generate_report.py` inside the `build_section_prompts()` function. Each section is a `(name, prompt)` tuple — modify the prompt text to change what Gemini searches for and how it writes.

### Add or remove sections

Add a new tuple to the list in `build_section_prompts()` and add a matching entry to the `nav_ids` list in `build_html_page()`.

### Change the AI model

Edit the `MODEL` constant at the top of `generate_report.py`:

```python
MODEL = "gemini-2.5-flash"   # Default — free, fast, strong quality
MODEL = "gemini-2.5-pro"     # More capable, may require paid tier
MODEL = "gemini-2.0-flash"   # Faster, slightly less detailed
```

### Send to multiple recipients

Change `TO_EMAIL` to a comma-separated list and update `send_email.py` to split and send to each.

### Change the sports teams

Edit the Sports section prompt in `build_section_prompts()` to replace Tottenham Hotspur and Nashville SC with your teams.

## Troubleshooting

**Workflow never runs on schedule:**
GitHub Actions cron jobs can be delayed by up to 15–30 minutes during high-demand periods. If the workflow hasn't run at all, make sure the repo has had recent activity (GitHub disables cron for inactive repos after 60 days).

**Email not received:**
- Check your spam/junk folder
- Verify the `GMAIL_ADDRESS`, `GMAIL_APP_PASSWORD`, and `TO_EMAIL` secrets are correct
- Make sure 2-Step Verification is enabled on the Gmail account
- Try regenerating the App Password

**Report sections show "could not be generated":**
- Check the workflow logs in the Actions tab for API errors
- Verify your `GEMINI_API_KEY` secret is valid
- The Gemini API may have had a transient error — re-run the workflow

**GitHub Pages shows 404:**
- Make sure you've enabled GitHub Pages with the `gh-pages` branch as the source
- The `gh-pages` branch is created on the first successful workflow run
- It can take a few minutes for the page to become available after the first deploy
